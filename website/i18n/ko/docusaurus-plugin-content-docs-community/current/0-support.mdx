# 지원

도큐사우루스 커뮤니티에는 수천 명의 개발자가 있습니다.

이 페이지에서는 도큐사우루스 관련 커뮤니티 몇 개를 소개합니다. 추가적인 온·오프라인 학습 자료는 다른 페이지를 참고하세요.

도큐사우루스 커뮤니티에 가입하기 전에 [활동 시 지켜야할 지침을 먼저 읽어주세요](https://engineering.fb.com/codeofconduct/). 우리는 [기여자 계약](https://www.contributor-covenant.org/)을 채택하고 있으며 모든 커뮤니티 구성원이 이를 지켜주기를 기대하고 있습니다.

## 스택 오버플로 {#stack-overflow}

스택 오버플로는 코드에 대한 질문이 생기거나 특정 오류를 만났을 때 자주 찾는 포럼 중 하나입니다. **docusaurus** 태그가 달린 [기존 질문](https://stackoverflow.com/questions/tagged/docusaurus)을 찾아서 읽어보거나 [새로운 질문](https://stackoverflow.com/questions/ask?tags=docusaurus)을 남겨보세요!

## 포럼에서 토론하기 {#discussion-forums}

도큐사우루스의 미래뿐 아니라 모범 사례, 애플리케이션 아키텍처에 대한 흥미로운 논의가 진행되는 온라인 포럼을 찾아볼 수 있습니다. 혹, 코드에 대한 질문이라면 스택 오버플로가 좀 더 유용할 겁니다.

- [도큐사우루스 온라인 채팅방](https://discord.gg/docusaurus)
  - [#help-and-questions](https://discord.gg/fwbcrQ3dHR) 사용자 지원 채팅방
  - [#contributors](https://discord.gg/6g6ASPA) 오픈소스 기여자 채팅방
- [레딧 도큐사우루스 커뮤니티](https://www.reddit.com/r/docusaurus/)

## 새로운 기능 요청 {#feature-requests}

새로운 기능 요청은 [feature requests board (Canny)](/feature-requests)에 남겨주세요. 로드맵을 위한 유용한 도구이며 추천 순으로 기능 요청 목록을 정렬할 수 있습니다. 깃헙 이슈에서는 어떤 요청에 대한 관심이 많은지 알 수 없지만, Canny board에서는 가장 많은 추천을 받은 요청을 개발 팀에서 빠르게 인지할 수 있습니다. 새로운 기능(특히 커다란 기능)은 풀 리퀘스트를 생성하지 말아주세요. 누군가 이미 만들고 있거나 로드맵의 일부일 수 있습니다. 새로운 기능이 필요하다면 우리에게 먼저 연락해주세요!
