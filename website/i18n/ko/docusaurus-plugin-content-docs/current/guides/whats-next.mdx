# 다음 단계는 뭔가요?

축하합니다! 이제 여러분은 도큐사우루스의 핵심적인 기능을 이해했습니다. 아래와 같은 기능들이죠.

- 독립형 리액트/마크다운 페이지를 만들기 위해 [페이지 플러그인을 사용했습니다](./creating-pages.mdx).
- 문서 페이지를 만들기 위해 [문서 플러그인 사용했습니다](./docs/docs-introduction.mdx). 여기에는 [사이드바 설정](./docs/sidebar/index.mdx)과 [버전 관리](./docs/versioning.mdx)를 포함합니다.
- 모든 기능을 갖춘 블로그를 만들기 위해 [블로그 플러그인 사용했습니다](../blog.mdx).
- 모든 콘텐츠 플러그인에 유용한 [다양한 마크다운 기능](./markdown-features/markdown-features-intro.mdx)도 직접 경험해보았습니다.
- 여러분의 사이트를 원하는 형태로 만들기 위해 [스타일 시트](../styling-layout.mdx)나 [스위즐링](../swizzling.mdx)도 사용해보았습니다.
- 여러분의 페이지에 [이미지나 기타 애셋도 추가해보았습니다](../static-assets.mdx).
- 사이트에 [검색 기능도 추가했습니다](../search.mdx).
- 기본 도큐사우루스 API를 통해 [브라우저 지원](../browser-support.mdx)과 [SEO](../seo.mdx)를 어떻게 처리할 수 있는지도 이해했습니다.
- [개별 플러그인](../using-plugins.mdx)을 설치하고 구성하는 방법을 배웠습니다.
- 콘텐츠 호스트에 여러분의 사이트를 [배포했습니다](../deployment.mdx).
- 여러분의 사이트에서 여러 언어를 지원하도록 [국제화](../i18n/i18n-tutorial.mdx)했습니다.

이제 여러분은 아마도 잘 구성된 `docusaurus.config.js` 파일을 이미😄 가지고 있을 겁니다. 하지만 아직 더 많은 코드를 작성할 수 있습니다! 대부분의 기능은 캡슐화된 도큐사우루스 API를 호출해서 구현됩니다. 여정을 계속하면서 세 가지 경로를 선택할 수 있습니다.

- 고급 도큐사우루스 개념에 대해 알아보세요. API가 어떻게 동작하는지 더 깊이 이해하는데 도움이 됩니다.
- [기존의 모든 API](../docusaurus-core.mdx)를 좀 더 살펴보세요. 아직 많은 부분이 가이드에서 다루어지지 못했습니다!
- 사이트의 기능을 확장하는 [플러그인을 개발](../api/plugin-methods/README.mdx)하는 방법을 배워봅니다.
