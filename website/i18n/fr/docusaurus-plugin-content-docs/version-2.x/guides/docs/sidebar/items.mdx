# Éléments de la barre latérale

```mdx-code-block
import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';
import BrowserWindow from '@site/src/components/BrowserWindow';
```

Nous avons introduit trois types d'éléments dans l'exemple dans la section précédente : `doc`, `category` et `link`, dont l'utilisation est assez intuitive. Nous présenterons explicitement leurs API. Il y a aussi un quatrième type : `autogenerated`, que nous expliquerons plus en détail plus tard.

- **[Doc](#sidebar-item-doc)** : lien vers une page de doc, l'associant à la barre latérale
- **[Link](#sidebar-item-link)** : lien vers une page interne ou externe
- **[Category](#sidebar-item-category)** : crée un menu déroulant des éléments de la barre latérale
- **[Autogenerated](autogenerated.mdx)** : génère automatiquement une barre latérale
- **[HTML](#sidebar-item-html)** : rend du HTML pur dans la position de l'élément
- **[\*Ref](multiple-sidebars.mdx#sidebar-item-ref)** : lien vers une page de doc, sans faire participer l'élément à la génération de la navigation

## Doc : lien vers un doc {#sidebar-item-doc}

Utilisez le type `doc` pour créer un lien vers une page doc et ajouter ce doc à une barre latérale :

```ts
type SidebarItemDoc =
  // Syntaxe normale
  | {
      type: 'doc';
      id : string;
      label: string; // Texte libellé de la barre latérale
      className?: string; // Nom de la classe pour le libellé de la barre latérale
      customProps?: Record<string, unknown>; // props personnalisés
    }

  // Syntaxe abrégée
  | string ; // raccourci docId
```

Exemple :

```js title="sidebars.js"
module.exports = {
  mySidebar: [
    // Syntaxe normale :
    // highlight-start
    {
      type: 'doc',
      id: 'doc1', // ID du document
      label: 'Pour commencer', // libellé de la barre latérale
    },
    // highlight-end

    // Syntaxe abrégée :
    // highlight-start
    'doc2', // ID du document
    // highlight-end
  ],
};
```

Si vous utilisez le raccourci de doc ou la barre latérale [autogenerated](autogenerated.mdx), vous perdrez la possibilité de personnaliser l'étiquette de la barre latérale à travers la définition de l'élément. Vous pouvez toutefois utiliser `sidebar_label` du front matter Markdown au sein de ce doc, qui a une priorité supérieure à la clé `label` dans l'élément de la barre latérale. De même, vous pouvez utiliser `sidebar_custom_props` pour déclarer des métadonnées personnalisées pour une page de doc.

:::note

Un élément `doc` définit une [association implicite de barre latérale](./multiple-sidebars.mdx#sidebar-association). N'affectez pas le même doc à plusieurs barres latérales : utilisez plutôt un `ref`.

:::

:::tip

Les props personnalisés de la barre latérale sont un moyen utile de propager des métadonnées arbitraires sur les documents du côté client, afin que vous puissiez obtenir des informations supplémentaires lorsque vous utilisez un hook lié aux documents qui récupère un objet doc.

:::

## Link : lien vers n'importe quelle page {#sidebar-item-link}

Utilisez le type `link` pour créer un lien vers une page (interne ou externe) qui n'est pas un doc.

```ts
type SidebarItemLink = {
  type: 'link';
  label: string;
  href: string;
  className?: string;
  description?: string;
};
```

Exemple :

```js title="sidebars.js"
module.exports = {
  myLinksSidebar: [
    // highlight-start
    // Lien externe
    {
      type: 'link',
      label: 'Facebook', // Le libellé du lien
      href: 'https://facebook.com', // L'URL externe
    },
    // highlight-end

    // highlight-start
    // Lien interne
    {
      type: 'link',
      label: 'Accueil', // Le libellé du lien
      href: '/', // Le chemin interne
    },
    // highlight-end
  ],
};
```

## HTML : rendre un balisage personnalisé {#sidebar-item-html}

Utilisez le type `html` pour afficher du HTML personnalisé dans la balise `<li>` de l'élément.

Cela peut être utile pour insérer des éléments personnalisés tels que des divisions, des titres de section, des publicités et des images.

```ts
type SidebarItemHtml = {
  type: 'html';
  value: string;
  defaultStyle?: boolean; // Utiliser les styles de l'élément du menu par défaut
  className?: string;
};
```

Exemple :

```js title="sidebars.js"
module.exports = {
  myHtmlSidebar: [
    // highlight-start
    {
      type: 'html',
      value: '<img src="sponsor.png" alt="Sponsor" />', // Le HTML à rendre
      defaultStyle: true, // Utilise le style par défaut des éléments du menu
    },
    // highlight-end
  ],
};
```

:::tip

L'élément du menu est déjà enveloppé dans une balise `<li>`, donc si votre élément personnalisé est simple, comme un titre, il suffit de fournir une chaîne comme valeur et d'utiliser la propriété `className` pour le styliser :

```js title="sidebars.js"
module.exports = {
  myHtmlSidebar: [
    {
      type: 'html',
      value: 'Concepts de base',
      className: 'sidebar-title',
    },
  ],
};
```

:::

## Category : créer une hiérarchie {#sidebar-item-category}

Utilisez le type `category` pour créer une hiérarchie des éléments de la barre latérale.

```ts
type SidebarItemCategory = {
  type: 'category';
  label: string; // Texte du libellé de la barre latérale.
  items: SidebarItem[]; // Tableau d'éléments de la barre latérale.
  className?: string;
  description?: string;

  // Options de catégorie :
  collapsible: boolean; // Configure la catégorie pour qu'elle soit repliable
  collapsed: boolean; // Configure la catégorie pour qu'elle soit initialement réduite ou ouverte par défaut
  link: SidebarItemCategoryLinkDoc | SidebarItemCategoryLinkGeneratedIndex;
};
```

Exemple :

```js title="sidebars.js"
module.exports = {
  docs: [
    {
      type: 'category',
      label: 'Guides',
      collapsible: true,
      collapsed: false,
      items: [
        'creating-pages',
        {
          type: 'category',
          label: 'Docs',
          items: ['introduction', 'sidebar', 'markdown-features', 'versioning'],
        },
      ],
    },
  ],
};
```

:::tip

Utilisez la [**syntaxe raccourci**](#category-shorthand) lorsque vous n'avez pas besoin de personnalisations :

```js title="sidebars.js"
module.exports = {
  docs: {
    Guides: [
      'creating-pages',
      {
        Docs: ['introduction', 'sidebar', 'markdown-features', 'versioning'],
      },
    ],
  },
};
```

:::

### Liens de catégorie {#category-link}

Avec les liens de catégorie, un clic sur une catégorie peut vous faire accéder à une autre page.

:::tip

Utilisez les liens de catégorie pour introduire une catégorie de documents.

Les catégories générées automatiquement peuvent utiliser le fichier [`_category_.yml`](./autogenerated.mdx#category-item-metadata) pour déclarer le lien.

:::

#### Page d'index générée {#generated-index-page}

Vous pouvez générer automatiquement une page d'index qui affiche tous les enfants directs de cette catégorie. Le `slug` vous permet de personnaliser la route de la page générée, qui par défaut est `/category/[categoryName]`.

```js title="sidebars.js"
module.exports = {
  docs: [
    {
      type: 'category',
      label: 'Guides',
      // highlight-start
      link: {
        type: 'generated-index',
        title: 'Guides de Docusaurus',
        description: 'En savoir plus sur les concepts Docusaurus les plus importants !',
        slug: '/category/docusaurus-guides',
        keywords: ['guides'],
        image: '/img/docusaurus.png',
      },
      // highlight-end
      items: ['pages', 'docs', 'blog', 'search'],
    },
  ],
};
```

Voyez-la en action dans la [page des Guides Docusaurus](/docs/category/guides).

:::tip

Utilisez les liens `generated-index` comme moyen rapide d'obtenir un document d'introduction.

:::

#### Lien de doc {#category-doc-link}

Une catégorie peut créer un lien vers un document existant.

```js title="sidebars.js"
module.exports = {
  docs: [
    {
      type: 'category',
      label: 'Guides',
      // highlight-start
      link: {type: 'doc', id: 'introduction'},
      // highlight-end
      items: ['pages', 'docs', 'blog', 'search'],
    },
  ],
};
```

Voyez-la en action sur la [page d'introduction i18n](../../../i18n/i18n-introduction.mdx).

#### Intégration de l'index généré dans une page de doc {#embedding-generated-index-in-doc-page}

Vous pouvez également intégrer la liste de cartes générée dans une page de doc ordinaire avec le composant `DocCardList`. Elle affichera tous les éléments de la barre latérale de la catégorie parente du document courant.

```md title="docs/sidebar/index.md"
import DocCardList from '@theme/DocCardList';

<DocCardList />
```

```mdx-code-block
<BrowserWindow>

import DocCardList from '@theme/DocCardList';

<DocCardList />

</BrowserWindow>
```

### Catégories repliables {#collapsible-categories}

Nous prenons en charge l'option permettant de développer/réduire les catégories. Les catégories sont repliables par défaut, mais vous pouvez désactiver le pliage avec `collapsible: false`.

```js title="sidebars.js"
module.exports = {
  docs: [
    {
      type: 'category',
      label: 'Guides',
      items: [
        'creating-pages',
        {
          type: 'category',
          // highlight-next-line
          collapsible: false,
          label: 'Docs',
          items: ['introduction', 'sidebar', 'markdown-features', 'versioning'],
        },
      ],
    },
  ],
};
```

Pour rendre toutes les catégories non pliables par défaut, définissez l'option `sidebarCollapsible` dans `plugin-content-docs` à `false` :

```js title="docusaurus.config.js"
module.exports = {
  presets: [
    [
      '@docusaurus/preset-classic',
      {
        docs: {
          // highlight-next-line
          sidebarCollapsible: false,
        },
      },
    ],
  ],
};
```

:::note

L'option dans `sidebars.js` a la priorité sur la configuration du plugin, il est donc possible de rendre certaines catégories pliables lorsque `sidebarCollapsible` est défini à `false` globalement.

:::

### Catégories développées par défaut {#expanded-categories-by-default}

Les catégories repliables sont réduites par défaut. Si vous voulez qu'elles soient développées au premier rendu, vous pouvez définir `collapsed` à `false` :

```js title="sidebars.js"
module.exports = {
  docs: {
    Guides: [
      'creating-pages',
      {
        type: 'category',
        label: 'Docs',
        // highlight-next-line
        collapsed: false,
        items: ['markdown-features', 'sidebar', 'versioning'],
      },
    ],
  },
};
```

Similaire à `collapsible`, vous pouvez également définir la configuration globale `options.sidebarCollapsed` à `false`. Les options individuelles `collapsed` dans `sidebars.js` auront toujours la priorité sur cette configuration.

```js title="docusaurus.config.js"
module.exports = {
  presets: [
    [
      '@docusaurus/preset-classic',
      {
        docs: {
          // highlight-next-line
          sidebarCollapsed: false,
        },
      },
    ],
  ],
};
```

:::warning

Lorsqu'une catégorie a `collapsed: true` mais `collapsible: false` (soit par `sidebars.js`, soit par la configuration du plugin), ce dernier prend le dessus et la catégorie est toujours rendue comme développée.

:::

## Utilisation de raccourcis {#using-shorthands}

Vous pouvez exprimer les éléments typiques de la barre latérale sans beaucoup de personnalisation de manière plus concise avec des **syntaxes de raccourci**. Il y a deux éléments à ce sujet : [**raccourcis de doc**](#doc-shorthand) et [**raccourcis de catégorie**](#category-shorthand).

### Raccourci de doc {#doc-shorthand}

Un élément de type `doc` peut être simplement une chaîne représentant son ID :

```mdx-code-block
<Tabs>
<TabItem value="Écriture courante">
```

```js title="sidebars.js"
module.exports = {
  sidebar: [
    // highlight-start
    {
      type: 'doc',
      id: 'myDoc',
    },
    // highlight-end
  ],
};
```

```mdx-code-block
</TabItem>
<TabItem value="Raccourci">
```

```js title="sidebars.js"
module.exports = {
  sidebar: [
    // highlight-start
    'myDoc',
    // highlight-end
  ],
};
```

```mdx-code-block
</TabItem>
</Tabs>
```

Il est donc possible de simplifier l'exemple ci-dessus ainsi :

```js title="sidebars.js"
module.exports = {
  mySidebar: [
    {
      type: 'category',
      label: 'Pour commencer',
      items: [
        // highlight-next-line
        'doc1',
      ],
    },
    {
      type: 'category',
      label: 'Docusaurus',
      items: [
        // highlight-start
        'doc2',
        'doc3',
        // highlight-end
      ],
    },
    {
      type: 'link',
      label: 'En savoir plus',
      href: 'https://example.com',
    },
  ],
};
```

### Raccourci de catégorie {#category-shorthand}

Un élément de catégorie peut être représenté par un objet dont la clé est son label, et la valeur est un tableau de sous-éléments.

```mdx-code-block
<Tabs>
<TabItem value="Écriture courante">
```

```js title="sidebars.js"
module.exports = {
  sidebar: [
    // highlight-start
    {
      type: 'category',
      label: 'Démarrage',
      items: ['doc1', 'doc2'],
    },
    // highlight-end
  ],
};
```

```mdx-code-block
</TabItem>
<TabItem value="Raccourci">
```

```js title="sidebars.js"
module.exports = {
  sidebar: [
    // highlight-start
    {
      'Démarrage': ['doc1', 'doc2'],
    },
    // highlight-end
  ],
};
```

```mdx-code-block
</TabItem>
</Tabs>
```

Cela nous permet de simplifier cet exemple en :

```js title="sidebars.js"
module.exports = {
  mySidebar: [
    // highlight-start
    {
      'Pour commencer': ['doc1'],
    },
    {
      Docusaurus: ['doc2', 'doc3'],
    },
    // highlight-end
    {
      type: 'link',
      label: 'En savoir plus',
      href: 'https://example.com',
    },
  ],
};
```

Chaque objet raccourci après cette transformation contiendra exactement une entrée. Considérez maintenant l'exemple simplifié ci-dessous :

```js title="sidebars.js"
module.exports = {
  mySidebar: [
    // highlight-start
    {
      'Pour commencer': ['doc1'],
      Docusaurus: ['doc2', 'doc3'],
    },
    // highlight-end
    {
      type: 'link',
      label: 'En savoir plus',
      href: 'https://example.com',
    },
  ],
};
```

Notez comment les deux catégories consécutives sont comprimées en un seul objet avec deux entrées. Cette syntaxe génère une **section de barre latérale** : vous ne devez pas voir cet objet comme un élément en vrac - cet objet est déplié, chaque entrée devenant un élément distinct, et ils sont raccordés ensemble avec le reste des éléments (dans ce cas, le lien « En savoir plus ») pour former le niveau final de la barre latérale. Les sections de barres latérales sont également importantes lorsqu'on parle de [barres latérales autogénérées](autogenerated.mdx).

Chaque fois que vous avez un tableau d'éléments qui est réduit à un arcccourci d'une catégorie, vous pouvez également omettre le tableau qui l'entoure.

```mdx-code-block
<Tabs>
<TabItem value="Avant">
```

```js title="sidebars.js"
module.exports = {
  sidebar: [
    {
      'Démarrage': ['doc1'],
      Docusaurus: [
        {
          'Guides de base': ['doc2', 'doc3'],
          'Guides avancés': ['doc4', 'doc5'],
        },
      ],
    },
  ],
};
```

```mdx-code-block
</TabItem>
<TabItem value="Après">
```

```js title="sidebars.js"
module.exports = {
  sidebar: {
    'Démarrage': ['doc1'],
    Docusaurus: {
      'Guides de base': ['doc2', 'doc3'],
      'Guides avancés': ['doc4', 'doc5'],
    },
  },
};
```

```mdx-code-block
</TabItem>
</Tabs>
```
