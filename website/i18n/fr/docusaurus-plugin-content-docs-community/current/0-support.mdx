# Support

Docusaurus a une communauté de milliers de développeurs.

Sur cette page, nous avons répertorié quelques communautés liées à Docusaurus dont vous pouvez faire partie. Consultez les autres pages de cette section pour trouver d'autres supports d'apprentissage en ligne et en personne.

Avant de participer aux communautés de Docusaurus, [veuillez lire notre code de conduite](https://engineering.fb.com/codeofconduct/). Nous avons adopté le [Contributor Covenant](https://www.contributor-covenant.org/) et nous nous attendons à ce que tous les membres de la communauté adhèrent aux directives qui s'y trouvent.

## Stack Overflow {#stack-overflow}

Stack Overflow est un forum populaire pour poser des questions reliées au code ou à des erreurs spécifiques. Lisez les [questions existantes](https://stackoverflow.com/questions/tagged/docusaurus) taggées **docusaurus** ou [posez vos propres questions](https://stackoverflow.com/questions/ask?tags=docusaurus) !

## Forums de discussion {#discussion-forums}

Il existe de nombreux forums en ligne pour discuter de meilleures pratiques et de l'architecture des applications ainsi que de l'avenir de Docusaurus. Si vous avez une question en rapport avec le code, il est généralement plus approprié de la poser dans Stack Overflow.

- [Chat en ligne de Docusaurus](https://discord.gg/docusaurus)
  - [#help-and-questions](https://discord.gg/fwbcrQ3dHR) pour l'aide de l'utilisateur
  - [#contributors](https://discord.gg/6g6ASPA) pour contribuer à l'aide
- [Communauté Docusaurus de Reddit](https://www.reddit.com/r/docusaurus/)

## Demandes de fonctionnalités {#feature-requests}

Pour de nouvelles demandes de fonctionnalités, vous pouvez créer un message sur notre [tableau de nouvelles demandes de fonctionnalités (Canny)](/feature-requests), qui est un outil pratique pour la feuille de route et permet de trier par vote positif, qui donne à l'équipe un meilleur indicateur des fonctionnalités qui sont en forte demande, par rapport aux problèmes GitHub qui sont plus difficiles à trier. Évitez de faire un Pull Request pour de nouvelles fonctionnalités (en particulier les plus grandes) car quelqu'un pourrait déjà y travailler ou fera déjà partie de notre feuille de route. Parlez-nous d'abord !

## Actualités {#news}

For the latest news about Docusaurus, [follow **@docusaurus** on X](https://x.com/docusaurus) and the [official Docusaurus blog](/blog) on this website.
