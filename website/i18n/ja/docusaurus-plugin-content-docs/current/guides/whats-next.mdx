# その後は？

おめでとうございます！ Docusaurus の主要な機能のほとんどを理解しました。 あなたは以下のことを成し遂げました:

- [pages プラグインを使って](./creating-pages.mdx)、スタンドアローンの React / Markdown ページを作成しました
- [docs プラグインを使って](./docs/docs-introduction.mdx)、ドキュメンテーションページを作成しました。 これには、[サイドバーの設定](./docs/sidebar/index.mdx)や[バージョン履歴](./docs/versioning.mdx)も含まれます。
- [blog プラグインを使って](../blog.mdx)、完全な機能を持つブログを作成しました
- すべてのコンテンツプラグインに便利な、[一連の Markdown 機能](./markdown-features/markdown-features-intro.mdx) を試してみました
- [スタイルシート](../styling-layout.mdx) または [スイズリング](../swizzling.mdx) を使って、サイトの外観をカスタマイズしました
- ページに [画像やその他の資産を組み込みました](../static-assets.mdx)
- サイトに [検索機能を追加しました](../search.mdx)
- Docusaurus の標準 API を通して [ブラウザー対応](../browser-support.mdx) や [SEO](../seo.mdx) を行う方法を理解しました
- [各プラグイン](../using-plugins.mdx) がどのようにしてインストール・設定されるのかを学びました
- サイトをコンテンツホストに [デプロイ](../deployment.mdx) しました
- サイトを [国際化](../i18n/i18n-tutorial.mdx) し、複数の言語を組み込みました

この時点で、`docusaurus.config.js` のサイズは大きくなっていることでしょう😄とはいえ、まださほどコードを書いていませんね！ ほとんどの機能はカプセル化されたDocusaurus APIを呼び出すことによって実装されます。 この旅を続けると、３つの道があります。

- Docusaurus のより高度な概念を学びます。 これにより、APIが何を行っているのかをより深く理解できます。
- [存在するすべての API](../docusaurus-core.mdx) について学びます。 その多くは、ガイドでは扱わなかった API です！
- サイトの機能を拡張するための [プラグインの開発](../api/plugin-methods/README.mdx) を学びます。
