# サポート

Docusaurus には何千もの開発者のコミュニティがあります。

このページでは、Docusaurus関連のコミュニティをいくつか紹介しています。その他のオンラインおよび対面学習資料については、このセクションの他のページを参照してください。

Docusaurus のコミュニティに参加する前に、[行動規範 (Code of Conduct)](https://engineering.fb.com/codeofconduct/)をお読みください。 私たちは [コントリビュータ契約](https://www.contributor-covenant.org/) を採用しており、すべてのコミュニティメンバーがガイドラインを遵守していることを期待しています。

## Stack Overflow {#stack-overflow}

コードレベルでの質問をしたい場合や、特定のエラーが発生した場合は、Stack Overflow を利用してみましょう。 **docusaurus**でタグ付けされた[既存の質問](https://stackoverflow.com/questions/tagged/docusaurus)を一読するか、[自分で質問してみましょう](https://stackoverflow.com/questions/ask?tags=docusaurus)！

## ディスカッションフォーラム {#discussion-forums}

Docusaurus の将来やベストプラクティス、アプリケーションアーキテクチャについて議論できるオンラインフォーラムがたくさんあります。 コードレベルで回答可能な質問がある場合は、Stack Overflow の方が適しているでしょう。

- [Docusaurus オンラインチャット](https://discord.gg/docusaurus)
  - 利用上の質問は [#help-and-questions](https://discord.gg/fwbcrQ3dHR)
  - コントリビューションに関する質問は [#contributors](https://discord.gg/6g6ASPA)
- [Reddit の Docusaurus コミュニティ](https://www.reddit.com/r/docusaurus/)

## 新しい機能の提案 {#feature-requests}

新機能のリクエストについては、[機能リクエストボード (Canny)](/feature-requests) に投稿してください。これはロードマップのための便利なツールで、投票数によるソートができるようになっています。トリアージが難しい Github での Issue と比べて、開発コアチームにとってどのような機能に対する需要が高いのかよりわかりやすくなります。 誰かがすでにそれに取り組んでいるか、ロードマップの一部になる可能性があるため、新しい機能（特に大きなもの）の Pull Request を作成することは避けてください。 まずはご相談ください！

## お知らせ {#news}

Docusaurus に関する最新情報は、[X の **@docusaurus**](https://x.com/docusaurus) をフォローして、この Web サイトの[Docusaurus 公式ブログ](/blog)を参照してください。
