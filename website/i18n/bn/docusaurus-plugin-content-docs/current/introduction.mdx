# পরিচিতি

⚡️ Docusaurus আপনাকে **অল্প সময়ে একটি সুন্দর ডকুমেন্টেশন সাইট** তৈরি করতে সাহায্য করবে।

💸 একটি কাস্টম টেক স্ট্যাক তৈরি করা ব্যয়বহুল। পরিবর্তে, **আপনার কন্টেন্টে মনোনিবেশ করুন** এবং শুধু Markdown ফাইল লিখুন।

💥 আরও প্রস্তুত? **অ্যাডভান্সড ফিচার** যেমন ভার্সনিং, i18n, সার্চ এবং থিম কাস্টমাইজেশন ব্যবহার করুন।

💅 অনুপ্রেরণার জন্য **[সেরা Docusaurus সাইটগুলি](/showcase?tags=favorite)** দেখুন।

🧐 Docusaurus একটি **স্ট্যাটিক-সাইট জেনারেটর**। এটি দ্রুত ক্লায়েন্ট-সাইড নেভিগেশন সহ একটি **সিঙ্গেল-পেজ অ্যাপ্লিকেশন** তৈরি করে, আপনার সাইটকে ইন্টারঅ্যাক্টিভ করতে **React** এর সম্পূর্ণ শক্তি ব্যবহার করে। এটি বক্স থেকেই **ডকুমেন্টেশন ফিচার** প্রদান করে কিন্তু **যেকোনো ধরনের সাইট** তৈরি করতে ব্যবহার করা যেতে পারে (ব্যক্তিগত ওয়েবসাইট, প্রোডাক্ট, ব্লগ, মার্কেটিং ল্যান্ডিং পেজ, ইত্যাদি)।

## ফাস্ট ট্র্যাক ⏱️ {/* #fast-track */}

খেলার মাধ্যমে **৫ মিনিটে** Docusaurus বুঝুন!

একটি নতুন Docusaurus সাইট তৈরি করুন এবং **খুব সংক্ষিপ্ত** এম্বেডেড টিউটোরিয়াল অনুসরণ করুন।

[Node.js](https://nodejs.org/en/download/) ইনস্টল করুন এবং একটি নতুন Docusaurus সাইট তৈরি করুন:

```bash
npx create-docusaurus@latest my-website classic
```

সাইট শুরু করুন:

```bash
cd my-website
npx docusaurus start
```

[`http://localhost:3000`](http://localhost:3000) খুলুন এবং টিউটোরিয়াল অনুসরণ করুন।

:::tip

আপনার ব্রাউজারে অবিলম্বে Docusaurus পরীক্ষা করতে **[docusaurus.new](https://docusaurus.new)** ব্যবহার করুন!

অথবা অনলাইনে **[৫-মিনিটের টিউটোরিয়াল](https://tutorial.docusaurus.io)** পড়ুন।

:::

## v1 থেকে মাইগ্রেট করা {/* #migrating-from-v1 */}

Docusaurus v2+ Docusaurus v1 থেকে সম্পূর্ণ পুনর্লিখন হয়েছে, একটি সম্পূর্ণ আধুনিক টুলচেইনের সুবিধা নিয়ে। [v2 এর অফিসিয়াল রিলিজের](https://docusaurus.io/blog/2022/08/01/announcing-docusaurus-2.0) পরে, আমরা দৃঢ়ভাবে আপনাকে **Docusaurus v1 এর পরিবর্তে Docusaurus v2+ ব্যবহার করতে** উৎসাহিত করি, কারণ Docusaurus v1 অবচিত হয়ে গেছে।

[অনেক ইউজার](/showcase) ইতিমধ্যে Docusaurus v2+ ব্যবহার করছেন ([ট্রেন্ডস](https://www.npmtrends.com/docusaurus-vs-@docusaurus/core))।

**Docusaurus v2+ ব্যবহার করুন যদি:**

- :white_check_mark: আপনি একটি আধুনিক Jamstack ডকুমেন্টেশন সাইট চান
- :white_check_mark: আপনি ক্লায়েন্ট-সাইড রাউটিং সহ একটি সিঙ্গেল-পেজ অ্যাপ্লিকেশন (SPA) চান
- :white_check_mark: আপনি React এবং MDX এর সম্পূর্ণ শক্তি চান
- :white_check_mark: আপনার IE11 এর জন্য সাপোর্ট প্রয়োজন নেই

**[Docusaurus v1](https://v1.docusaurus.io/) ব্যবহার করুন যদি:**

- :x: আপনি একটি সিঙ্গেল-পেজ অ্যাপ্লিকেশন (SPA) চান না
- :x: আপনার IE11 এর জন্য সাপোর্ট প্রয়োজন (...আপনার কি? IE [ইতিমধ্যে এন্ড-অফ-লাইফে পৌঁছেছে](https://docs.microsoft.com/en-us/lifecycle/products/internet-explorer-11) এবং আর অফিসিয়ালি সাপোর্ট করা হয় না)

বিদ্যমান v1 ইউজারদের জন্য যারা v2+ এ আপগ্রেড করতে চান, আপনি আমাদের [মাইগ্রেশন গাইড](./migration/index.mdx) অনুসরণ করতে পারেন।

## ফিচার {/* #features */}

Docusaurus ডেভেলপার এবং কন্ট্রিবিউটর এক্সপেরিয়েন্সের প্রতি উচ্চ মনোযোগ দিয়ে তৈরি করা হয়েছে।

- ⚛️ **💚 এবং React দিয়ে তৈরি**:
  - React দিয়ে প্রসারিত এবং কাস্টমাইজ করুন
  - আপনার নিজস্ব React কম্পোনেন্ট প্রদান করে আপনার সাইটের ব্রাউজিং এক্সপেরিয়েন্সের সম্পূর্ণ নিয়ন্ত্রণ পান
- 🔌 **প্লাগযোগ্য**:
  - একটি বেসিক টেমপ্লেট দিয়ে আপনার সাইট বুটস্ট্র্যাপ করুন, তারপর অ্যাডভান্সড ফিচার এবং প্লাগইন ব্যবহার করুন
  - কমিউনিটির সাথে শেয়ার করতে আপনার প্লাগইন ওপেন সোর্স করুন
- ✂️ **ডেভেলপার এক্সপেরিয়েন্স**:
  - এখনই আপনার ডক্স লেখা শুরু করুন
  - কন্ট্রিবিউটরদের দ্বারা আরও রক্ষণাবেক্ষণযোগ্য করতে ইউনিভার্সাল কনফিগারেশন এন্ট্রি পয়েন্ট
  - পরিবর্তনে বিদ্যুৎ-দ্রুত ইনক্রিমেন্টাল বিল্ড সহ হট রিলোডিং
  - রুট-ভিত্তিক কোড এবং ডেটা স্প্লিটিং
  - সহজেই GitHub Pages, Netlify, Vercel এবং অন্যান্য ডিপ্লয়মেন্ট সার্ভিসে পাবলিশ করুন

আমাদের শেয়ার করা লক্ষ্য—আপনার ইউজারদের দ্রুত তাদের প্রয়োজনীয় জিনিস খুঁজে পেতে এবং আপনার প্রোডাক্ট আরও ভালোভাবে বুঝতে সাহায্য করা। আমরা আপনার ডক্স সাইট সঠিক এবং ভালোভাবে তৈরি করতে সাহায্য করার জন্য আমাদের সেরা প্র্যাকটিস শেয়ার করি।

- 🎯 **SEO বান্ধব**:
  - প্রতিটি সম্ভাব্য পাথের জন্য HTML ফাইল স্ট্যাটিকভাবে জেনারেট করা হয়।
  - আপনার ইউজারদের তাদের হাতের সমস্যার সাথে সরাসরি সম্পর্কিত আপনার অফিসিয়াল ডক্সে পৌঁছাতে সাহায্য করার জন্য পেজ-নির্দিষ্ট SEO।
- 📝 **MDX দ্বারা চালিত**:
  - Markdown এ এম্বেড করা JSX এবং React এর মাধ্যমে ইন্টারঅ্যাক্টিভ কম্পোনেন্ট লিখুন।
  - আপনার ইউজারদের আপনার প্রোডাক্ট পছন্দ করাতে লাইভ এডিটরে আপনার কোড শেয়ার করুন।
- 🔍 **সার্চ**: আপনার সম্পূর্ণ সাইট সার্চযোগ্য।
- 💾 **ডকুমেন্ট ভার্সনিং**: প্রজেক্ট রিলিজের সাথে ডকুমেন্টেশন সিঙ্ক রাখতে সাহায্য করে।
- 🌍 **আন্তর্জাতিকীকরণ (i18n)**: একাধিক লোকেলে আপনার সাইট অনুবাদ করুন।

Docusaurus v2+ আপনার সমস্ত ইউজারদের জন্য সহানুভূতিশীলভাবে অ্যাক্সেসিবল এবং বিদ্যুৎ-দ্রুত হওয়ার জন্য জন্মগ্রহণ করেছে।

- ⚡️ **বিদ্যুৎ-দ্রুত**। Docusaurus v2+ [PRPL Pattern](https://developers.google.com/web/fundamentals/performance/prpl-pattern/) অনুসরণ করে যা নিশ্চিত করে যে আপনার কন্টেন্ট অত্যন্ত দ্রুত লোড হয়।
- 🦖 **অ্যাক্সেসিবল**। অ্যাক্সেসিবিলিটির প্রতি মনোযোগ, আপনার সাইটকে সমস্ত ইউজারদের জন্য সমানভাবে অ্যাক্সেসিবল করে তোলে।

## ডিজাইন নীতি {/* #design-principles */}

- **শেখার জন্য সামান্য**। Docusaurus শেখা এবং ব্যবহার করা সহজ হওয়া উচিত কারণ API বেশ ছোট। বেশিরভাগ জিনিস এখনও ইউজারদের দ্বারা অর্জনযোগ্য হবে, এমনকি যদি তাদের আরও কোড এবং লিখতে আরও সময় লাগে। অ্যাবস্ট্রাকশন না থাকা ভুল অ্যাবস্ট্রাকশন থাকার চেয়ে ভালো, এবং আমরা চাই না ইউজারদের ভুল অ্যাবস্ট্রাকশনের চারপাশে হ্যাক করতে হোক। বাধ্যতামূলক টক—[Minimal API Surface Area](https://www.youtube.com/watch?v=4anAwXYqLG8)।
- **স্বজ্ঞাত**। একটি Docusaurus প্রজেক্টের প্রজেক্ট ডিরেক্টরি দেখে বা নতুন ফিচার যোগ করার সময় ইউজাররা অভিভূত বোধ করবে না। এটি স্বজ্ঞাত এবং তার উপরে তৈরি করা সহজ দেখাবে, তারা যে পদ্ধতির সাথে পরিচিত তা ব্যবহার করে।
- **স্তরযুক্ত আর্কিটেকচার**। আমাদের স্ট্যাকের প্রতিটি স্তরের মধ্যে উদ্বেগের বিচ্ছেদ (কন্টেন্ট/থিমিং/স্টাইলিং) স্পষ্ট হওয়া উচিত—ভালভাবে অ্যাবস্ট্র্যাক্টেড এবং মডুলার।
- **সংবেদনশীল ডিফল্ট**। সাধারণ এবং জনপ্রিয় পারফরম্যান্স অপটিমাইজেশন এবং কনফিগারেশন ইউজারদের জন্য করা হবে কিন্তু তাদের সেগুলি ওভাররাইড করার বিকল্প দেওয়া হয়।
- **কোনো ভেন্ডর লক-ইন নেই**। ইউজারদের ডিফল্ট প্লাগইন বা CSS ব্যবহার করতে হবে না, যদিও তাদের দৃঢ়ভাবে উৎসাহিত করা হয়। React Loadable এবং React Router এর মতো নির্দিষ্ট কোর ইনফ্রাস্ট্রাকচার অদলবদল করা যায় না কারণ আমরা তাদের উপর ডিফল্ট পারফরম্যান্স অপটিমাইজেশন করি, কিন্তু উচ্চ-স্তরের নয়। Markdown ইঞ্জিন, CSS ফ্রেমওয়ার্ক, CSS পদ্ধতি এবং অন্যান্য আর্কিটেকচারের পছন্দ সম্পূর্ণভাবে ইউজারদের উপর নির্ভর করবে।

আমরা বিশ্বাস করি যে, ডেভেলপার হিসাবে, একটি লাইব্রেরি কীভাবে কাজ করে তা জানা আমাদের এটি ব্যবহার করতে আরও ভালো হতে সাহায্য করে। তাই আমরা Docusaurus এর আর্কিটেকচার এবং বিভিন্ন কম্পোনেন্ট ব্যাখ্যা করার জন্য প্রচেষ্টা উৎসর্গ করছি এই আশায় যে এটি পড়া ইউজাররা টুলটির গভীর বোঝাপড়া অর্জন করবে এবং এটি ব্যবহার করতে আরও দক্ষ হবে।

## অন্যান্য টুলের সাথে তুলনা {/* #comparison-with-other-tools */}

সমস্ত স্ট্যাটিক সাইট জেনারেটরের মধ্যে, Docusaurus এর ডকুমেন্টেশন সাইটের উপর একটি অনন্য ফোকাস রয়েছে এবং বক্স থেকেই অনেক ফিচার রয়েছে।

আমরা অন্যান্য প্রধান স্ট্যাটিক সাইট জেনারেটরও অধ্যয়ন করেছি এবং তুলনার উপর আমাদের অন্তর্দৃষ্টি শেয়ার করতে চাই, আশা করি আপনাকে সেখানে থাকা প্রিজম্যাটিক পছন্দগুলির মধ্যে নেভিগেট করতে সাহায্য করবে।

### Gatsby {/* #gatsby */}

[Gatsby](https://www.gatsbyjs.com/) অনেক ফিচার দিয়ে প্যাক করা, প্লাগইনের একটি সমৃদ্ধ ইকোসিস্টেম রয়েছে এবং Docusaurus যা করে তার সবকিছু করতে সক্ষম। স্বাভাবিকভাবেই, এটি একটি উচ্চতর লার্নিং কার্ভের খরচে আসে। Gatsby অনেক কিছু ভালোভাবে করে এবং অনেক ধরনের ওয়েবসাইট তৈরির জন্য উপযুক্ত। অন্যদিকে, Docusaurus একটি জিনিস অসাধারণভাবে ভালো করার চেষ্টা করে - কন্টেন্ট লেখা এবং পাবলিশ করার জন্য সেরা টুল হওয়া।

GraphQL Gatsby এর জন্যও বেশ কোর, যদিও আপনার অগত্যা একটি Gatsby সাইট তৈরি করতে GraphQL এর প্রয়োজন নেই। বেশিরভাগ ক্ষেত্রে যখন স্ট্যাটিক ওয়েবসাইট তৈরি করা হয়, আপনার GraphQL যে নমনীয়তা প্রদান করে তার প্রয়োজন হবে না।

Docusaurus v2+ এর অনেক দিক Gatsby সম্পর্কে সেরা জিনিসগুলি দ্বারা অনুপ্রাণিত হয়েছিল এবং এটি একটি দুর্দান্ত বিকল্প।

[Docz](https://github.com/pedronauck/docz) একটি Gatsby থিম যা ডকুমেন্টেশন ওয়েবসাইট তৈরি করতে। এটি বর্তমানে Docusaurus এর চেয়ে কম ফিচারযুক্ত।

### Next.js {/* #nextjs */}

[Next.js](https://nextjs.org/) আরেকটি খুব জনপ্রিয় হাইব্রিড React ফ্রেমওয়ার্ক। এটি আপনাকে একটি ভালো ডকুমেন্টেশন ওয়েবসাইট তৈরি করতে সাহায্য করতে পারে, কিন্তু এটি ডকুমেন্টেশন ইউজ-কেসের দিকে মতামতযুক্ত নয়, এবং Docusaurus বক্স থেকেই যা প্রদান করে তা বাস্তবায়ন করতে অনেক বেশি কাজের প্রয়োজন হবে।

[Nextra](https://github.com/shuding/nextra) Next.js এর উপরে তৈরি একটি মতামতযুক্ত স্ট্যাটিক সাইট জেনারেটর। এটি বর্তমানে Docusaurus এর চেয়ে কম ফিচারযুক্ত।

### VitePress {/* #vitepress */}

[VitePress](https://vitepress.dev/) এর Docusaurus এর সাথে অনেক মিল রয়েছে - উভয়ই কন্টেন্ট-কেন্দ্রিক ওয়েবসাইটের উপর ব্যাপকভাবে ফোকাস করে এবং বক্স থেকেই কাস্টমাইজড ডকুমেন্টেশন ফিচার প্রদান করে। তবে, VitePress Vue দ্বারা চালিত, যেখানে Docusaurus React দ্বারা চালিত। আপনি যদি একটি Vue-ভিত্তিক সমাধান চান, VitePress একটি ভালো পছন্দ হবে।

### MkDocs {/* #mkdocs */}

[MkDocs](https://www.mkdocs.org/) একটি জনপ্রিয় Python স্ট্যাটিক সাইট জেনারেটর যার মূল্য প্রস্তাব Docusaurus এর অনুরূপ।

আপনার যদি একটি সিঙ্গেল-পেজ অ্যাপ্লিকেশনের প্রয়োজন না হয় এবং React ব্যবহার করার পরিকল্পনা না থাকে তবে এটি একটি ভালো বিকল্প।

[Material for MkDocs](https://squidfunk.github.io/mkdocs-material/) একটি সুন্দর থিম।

### Docsify {/* #docsify */}

[Docsify](https://docsify.js.org/) একটি ডকুমেন্টেশন ওয়েবসাইট তৈরি করা সহজ করে, কিন্তু এটি একটি স্ট্যাটিক-সাইট জেনারেটর নয় এবং SEO বান্ধব নয়।

### GitBook {/* #gitbook */}

[GitBook](https://www.gitbook.com/) এর একটি খুব পরিষ্কার ডিজাইন রয়েছে এবং অনেক ওপেন সোর্স প্রজেক্ট দ্বারা ব্যবহৃত হয়েছে। একটি ওপেন-সোর্স টুলের পরিবর্তে একটি বাণিজ্যিক প্রোডাক্টের দিকে এর ফোকাস স্থানান্তরিত হওয়ার সাথে, এর অনেক প্রয়োজনীয়তা আর ওপেন সোর্স প্রজেক্টের ডকুমেন্টেশন সাইটের চাহিদার সাথে খাপ খায় না। ফলস্বরূপ, অনেকে অন্যান্য প্রোডাক্টের দিকে ঝুঁকেছে। আপনি Redux এর Docusaurus এ স্যুইচ সম্পর্কে [এখানে](https://github.com/reduxjs/redux/issues/3161) পড়তে পারেন।

বর্তমানে, GitBook শুধুমাত্র ওপেন-সোর্স এবং নন-প্রফিট টিমের জন্য বিনামূল্যে। Docusaurus সবার জন্য বিনামূল্যে।

### Jekyll {/* #jekyll */}

[Jekyll](https://github.com/jekyll/jekyll) সবচেয়ে পরিপক্ক স্ট্যাটিক সাইট জেনারেটরগুলির মধ্যে একটি এবং ব্যবহার করার জন্য একটি দুর্দান্ত টুল হয়েছে — আসলে, Docusaurus এর আগে, Facebook এর বেশিরভাগ ওপেন সোর্স ওয়েবসাইট Jekyll এ তৈরি! শুরু করা অত্যন্ত সহজ। আমরা Jekyll দিয়ে একটি স্ট্যাটিক সাইট তৈরি করার মতো একই ডেভেলপার এক্সপেরিয়েন্স আনতে চাই।

স্ট্যাটিকভাবে জেনারেট করা HTML এবং `<script />` ট্যাগ ব্যবহার করে যোগ করা ইন্টারঅ্যাক্টিভিটির সাথে তুলনায়, Docusaurus সাইটগুলি React অ্যাপ। আধুনিক JavaScript ইকোসিস্টেম টুলিং ব্যবহার করে, আমরা ডক্স সাইটের পারফরম্যান্স, অ্যাসেট বিল্ডিং পাইপলাইন এবং অপটিমাইজেশন এবং সেট আপ করার সহজতার উপর নতুন মান স্থাপন করার আশা করি।

### Rspress {/* #rspress */}

[Rspress](https://rspress.dev/) Rspack এর উপর ভিত্তি করে একটি দ্রুত স্ট্যাটিক সাইট জেনারেটর, একটি Rust-ভিত্তিক বান্ডলার। এটি MDX (React কম্পোনেন্ট সহ Markdown) দিয়ে কন্টেন্ট লেখা, ইন্টিগ্রেটেড টেক্সট সার্চ, মাল্টিলিঙ্গুয়াল সাপোর্ট (i18n), এবং প্লাগইনের মাধ্যমে এক্সটেনসিবিলিটি সাপোর্ট করে। মার্জিত ডকুমেন্টেশন এবং স্ট্যাটিক ওয়েবসাইট তৈরির জন্য ডিজাইন করা, Rspress স্ট্যাটিক HTML ফাইল তৈরি করে যা ডিপ্লয় করা সহজ।

Rspress এবং Docusaurus বেশ একই রকম। তাদের উভয়েরই তাদের সুবিধা এবং অসুবিধা রয়েছে। Rspress আরও সম্প্রতি তৈরি করা হয়েছে এবং একটি আধুনিক ইনফ্রাস্ট্রাকচার থেকে উপকৃত হয় যা দ্রুত সাইট বিল্ড সক্ষম করে। Docusaurus এর পরিপক্কতা, ব্যাপক ফিচার সেট, নমনীয়তা এবং শক্তিশালী কমিউনিটির জন্য দাঁড়িয়ে আছে। এটি পারফরম্যান্সের ক্ষেত্রে প্রতিযোগিতামূলক থাকার জন্য নিয়মিতভাবে [তার ইনফ্রাস্ট্রাকচার আধুনিকীকরণ](https://github.com/facebook/docusaurus/issues/10556) করছে।

## কিছু মিসিং? {/* #something-missing */}

আপনি যদি ডকুমেন্টেশনে সমস্যা খুঁজে পান বা সাধারণভাবে ডকুমেন্টেশন বা প্রজেক্ট কীভাবে উন্নত করা যায় সে সম্পর্কে পরামর্শ থাকে, তাহলে আমাদের জন্য একটি [ইস্যু ফাইল করুন](https://github.com/facebook/docusaurus), অথবা [@docusaurus](https://x.com/docusaurus) X অ্যাকাউন্ট উল্লেখ করে একটি টুইট পাঠান।

নতুন ফিচার রিকোয়েস্টের জন্য, আপনি আমাদের [ফিচার রিকোয়েস্ট বোর্ড (Canny)](/feature-requests) এ একটি পোস্ট তৈরি করতে পারেন, যা রোড-ম্যাপিংয়ের জন্য একটি সহজ টুল এবং আপভোট দ্বারা সাজানোর অনুমতি দেয়, যা কোর টিমকে কোন ফিচারগুলি উচ্চ চাহিদায় রয়েছে তার একটি ভালো সূচক দেয়, GitHub ইস্যুর তুলনায় যা ট্রাইয়েজ করা কঠিন। নতুন ফিচারের জন্য (বিশেষত বড়গুলির) পুল রিকোয়েস্ট তৈরি করা থেকে বিরত থাকুন কারণ কেউ ইতিমধ্যে এটিতে কাজ করছে বা এটি আমাদের রোডম্যাপের অংশ হবে। প্রথমে আমাদের সাথে কথা বলুন!
