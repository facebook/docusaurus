# 何去何从？

恭喜！ 你已经理解了 Docusaurus 的大部分核心功能了。 你已经完成了这些工作：

- [使用页面插件](./creating-pages.mdx) 创建了一个独立的 React / Markdown页面
- [使用文档插件](./docs/docs-introduction.mdx)创建了一些文档页面。 这包括了[配置侧边栏](./docs/sidebar/index.mdx)，甚至是[版本化](./docs/versioning.mdx)
- [使用博客插件](../blog.mdx)创建了一个功能繁多的博客
- 尝试了一系列在所有内容插件都通用的 [Markdown 特性](./markdown-features/markdown-features-intro.mdx)
- [使用样式表](../styling-layout.mdx)或者 [swizzle](../swizzling.mdx) 个性化定制了你的网站的外观
- 在你的页面中[加入了一些图片和其他资源](../static-assets.mdx)
- 为你的网站[添加了搜索功能](../search.mdx)
- 了解了[浏览器支持](../browser-support.mdx)和[搜索引擎优化](../seo.mdx)是如何通过标准 Docusaurus API 实现的
- 学习了[单个插件](../using-plugins.mdx)是如何安装和配置的
- 把你的网站[部署](../deployment.mdx)到了某个托管商上
- 把你的网站[国际化](../i18n/i18n-tutorial.mdx)，让它包含了多门语言

到了这时候，你大概率已经有一个长长的 `docusaurus.config.js` 了😄 然而，你还没写过多少代码呢！ 大多数功能都是通过调用封装好的 Docusaurus API 实现的。 接下来，要继续你的旅程，面前有三条路：

- 学习更高级的 Docusaurus 概念。 这会有助于你更深入地理解这些 API 是做什么的。
- 阅读关于[所有现有的 API](../docusaurus-core.mdx) 的文档。 其中许多还没有在教程中涵盖到呢！
- 学习[开发一个插件](../api/plugin-methods/README.mdx)来扩展你的网站的功能。
